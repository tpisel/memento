# Writing summaries

Lead with the answer, not the question.

A summary is read by an agent scanning `memento brief` to decide whether opening the full note is worth the context. If the summary states the decision, constraint, or load-bearing fact, the agent can often act without opening the note. If the summary only says what topic the note discusses, the agent has to open every plausible match to discover the point.

## Heuristics

- Put the decision, invariant, or durable finding in the first sentence.
- Prefer specific paths, flags, version boundaries, and scope limits over category labels.
- "Does X", "supports Y", and "rejects Z" are usually stronger than "discusses", "considers", or "explores".
- Keep rejected alternatives and motivation in the body unless they are themselves the load-bearing fact.
- Length is secondary. A three-sentence summary that answers the retrieval question beats a short summary that only names the topic.

## Check

After writing a summary, ask: if this were the only line shown in `memento brief`, would an agent know why this note matters and whether to open it? If not, revise toward the answer.
